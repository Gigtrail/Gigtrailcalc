Replit prompt:

Refactor the Trail / Tour logic in Gig Trail so it behaves like a real touring run planner.

GOAL:
Make the Trail page smarter, more accurate, and easier to trust by improving:
1. date validation
2. drive time / distance between stops
3. blank days between shows
4. route logic
5. summary clarity
6. calculation reliability

==================================================
1. EVENT DATES MUST FALL WITHIN TOUR DATES
==================================================

Add validation so every stop/show date must fall within the tour start and end dates.

Rules:
- Each stop date must be:
  - on or after the tour start date
  - on or before the tour end date

If a stop falls outside the tour range:
- show inline validation error
- prevent save until fixed

Suggested error text:
- "This stop falls outside the selected tour dates."

Also:
- if the user changes the overall tour dates later, re-check all existing stops
- show a warning if any stops are now invalid

==================================================
2. SHOW DRIVE DISTANCE + DRIVE TIME BETWEEN STOPS
==================================================

For every segment in the trail, calculate and display:

- distance between locations
- estimated drive time

Examples:
- Start → Stop 1
- Stop 1 → Stop 2
- Stop 2 → Stop 3
- Final Stop → Return Home

Display format example:
- "356 km • 4 hr 12 min"
- or
- "356 km / 4h 12m"

Use:
- actual coordinates if available
- fallback to location text if needed

If route lookup fails:
- show:
  - "Drive time unavailable"
  - allow manual distance override if already supported

3. AUTO-RECOGNISE BLANK DAYS BETWEEN SHOWS
==================================================

Add support for blank days between stops so the tour reflects real downtime between shows.

Definition:
- A blank day is a day within the tour date range where there is no show booked.

Requirements:
- Detect gaps between show dates automatically
- Show blank days visually in the trail between stops

Example:
- Show on Apr 19
- next show on Apr 21
- Apr 20 should appear as a blank day between them

Suggested display:
- "Blank Day"
- date shown clearly
- optional subtitle like:
  - "Travel / Rest Day"
  - "No show booked"

Blank days should:
- count toward total Days on Tour
- count toward accommodation nights unless otherwise excluded later
- help users see pacing in the run

Keep this lightweight:
- do not make blank days full show cards
- show them as simple inserted timeline rows/cards between stops
