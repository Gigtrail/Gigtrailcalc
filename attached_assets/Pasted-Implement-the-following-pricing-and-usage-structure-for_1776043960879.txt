Implement the following pricing and usage structure for Gig Trail.

## Goal
Set a clearer free vs Pro model that feels fair to users, protects API usage, and positions Gig Trail as a serious touring tool.

We want:
- Free plan = useful but limited
- Pro plan = clear value upgrade
- pricing = $12/month or $79/year

---

## 1. Pricing structure

Update the app’s pricing model to:

### Free
- 5 calculations per week

### Pro
- $12/month
- $79/year

Where pricing is shown, make sure the yearly plan is clearly presented as the best value option.

### Recommended yearly helper copy:
- Best value
- Save 45%
- Less than $7/month

Use whichever feels cleanest in the UI.

---

## 2. Free usage limit

Change the free plan usage allowance from 10 calculations to:

- 5 calculations per week

This weekly limit should apply to the Single Show Calculator / show calculations.

### Requirements:
- track number of calculations used this week
- reset automatically each week
- show remaining usage clearly where appropriate
- do not break existing calculation logic

### Example copy:
- 1 of 5 free calculations used this week
- 5 of 5 free calculations used this week

---

## 3. Weekly reset behavior

Implement weekly reset logic for free calculations.

### Requirements:
- usage should reset every 7 days or on a defined weekly cycle
- keep implementation simple and reliable
- if there is already an existing usage tracking system, update it instead of rebuilding everything from scratch

Store enough data to know:
- current usage count
- reset date / week window

---

## 4. Pro entitlements

When a user upgrades to Pro, unlock:

- unlimited calculations
- Garage Pro features
- multiple vehicles
- assign vehicles to band members
- accommodation automation / custom limits where applicable
- full calculator access
- future Pro-only advanced logistics features

Do not hardcode only the current features if the app already has a feature-flag structure.
If feature gating exists, update it cleanly.

---

## 12. Product intent

This pricing should position Gig Trail as:
- affordable
- useful
- serious
- built for working musicians

It should not feel like a cheap gimmick.

The app is not just a calculator.
It is a decision tool to help musicians avoid bad gigs and plan better tours.

That positioning should come through in upgrade messaging and pricing UI.

---

## 13. Implementation priorities

Please implement in this order:

1. Change free usage from 10 to 5 calculations per week
2. Add / confirm weekly reset logic
3. Update upgrade banners and calculator usage text
4. Update all pricing displays to $12/month and $79/year
5. Emphasise yearly plan in pricing UI

## 14. UX rule

The upgrade flow should feel like:
- a smart next step
- unlocking better planning tools

- a punishment for using the app
