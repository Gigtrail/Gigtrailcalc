# Feature: Ticket Recovery (Break-Even Engine)

## Goal

Add a **Ticket Recovery** feature to the Tour Calc for mixed tours where some shows are:

* guaranteed fee
* ticketed
* door split
* blank travel days

The purpose is:

👉 If the full tour is currently losing money, show how many tickets need to be sold across the ticketed shows to recover the loss.

This must work even when ticket prices vary between shows.

❗ Do NOT use one averaged ticket price across the whole tour.
👉 Calculate recovery per show.

---

Inspect and reuse:

* stop / show form
* tour calculator logic
* results summary

Identify:

* how revenue per show is stored
* how show type is represented (if at all)
* where expenses are calculated
* where final net result is calculated
* whether ticket price / capacity already exist

Adapt this feature to the current structure — do not rebuild the system unnecessarily.

---

## When to show Ticket Recovery

### Show only if:

* `tourNetResult < 0`
* AND at least one ticketed show exists

### If profitable:

Show:

* `Tour is already profitable`
* `Ticketed shows will increase profit further`

### If no ticketed shows:

Show:

* `No ticketed shows available to recover this loss`

---

## Core calculation

Use existing final result:

```ts
tourNetResult = grossIncome - totalExpenses
```

If negative:

```ts
deficit = Math.abs(tourNetResult)
```

---

## Ticketed show requirements

Each ticketed show should have (map to existing or add):

* ticketPrice
* capacity (optional but preferred)
* perTicketFees (optional)
* perTicketCosts (optional)
* showName

If fields don’t exist, add minimal required fields.

---

## Per-show net ticket value

For each ticketed show:

```ts
netPerTicket = ticketPrice - perTicketFees - perTicketCosts
```

Defaults:

* perTicketFees = 0
* perTicketCosts = 0

If `netPerTicket <= 0`, exclude the show from recovery calculations.

---

## Weighted recovery model (IMPORTANT)

Do NOT split deficit evenly.

Use weighted distribution:

```ts
totalRecoveryWeight = sum(netPerTicket across valid shows)

showWeight = netPerTicket / totalRecoveryWeight

showDeficitShare = deficit * showWeight

ticketsNeeded = Math.ceil(showDeficitShare / netPerTicket)
```

---

## Capacity %

If capacity exists:

```ts
capacityPercentNeeded = ticketsNeeded / capacity
```

Display as:

* 32%
* 41%

If no capacity:

* show tickets needed
* display `Capacity unknown`

---

## Maximum recovery check

If capacity exists:

```ts
maxRecovery = netPerTicket * capacity
```

If total max recovery across all shows < deficit:

Show:

* `Even at full capacity, current ticketed shows cannot fully recover this loss`

---

## Strongest recovery show

Find show with highest `netPerTicket`.

Display:

* `Best recovery show: {showName}`
