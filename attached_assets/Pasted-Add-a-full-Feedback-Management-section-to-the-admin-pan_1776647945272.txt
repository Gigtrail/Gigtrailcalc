Add a full Feedback Management section to the admin panel.

Goal:
Create an admin-only feedback management system so the owner/admin can:
- view all feedback posts
- search/filter/sort feedback
- update status/category
- add internal admin notes (private, admin-only)
- add a public admin reply (visible to users on the feedback page)
- delete feedback posts
- optionally soft delete instead of hard delete
- keep the current public feedback board working

Important:
- Do not break existing voting
- Do not break existing feedback creation
- Respect current admin auth and permanent admin protections
- Reuse existing UI patterns where possible
- Keep this clean and production-ready, not a hack

==================================================
1. BACKEND / DATABASE
==================================================

Inspect the current feedback schema/model and extend it safely.

Add fields to feedback posts if not already present:
- adminReply: text nullable
- adminReplyUpdatedAt: datetime nullable
- internalNotes: text nullable
- deletedAt: datetime nullable (for soft delete)
- deletedByUserId: text nullable

If a separate admin notes table is cleaner, that is acceptable, but simplest is fine for alpha.

Also add timestamps if missing:
- createdAt
- updatedAt

Ensure existing rows still work after migration.

==================================================
2. API ROUTES
==================================================

Add admin-only feedback routes under /api/admin/feedback

Required endpoints:

1. GET /api/admin/feedback
- return all non-deleted feedback
- support query params:
  - search
  - status
  - category
  - sort (newest, oldest, top_voted)
- include:
  - id
  - title
  - description
  - category
  - status
  - votes
  - createdBy / author if available
  - createdAt
  - updatedAt
  - adminReply
  - internalNotes
  - any useful metadata

2. GET /api/admin/feedback/:id
- return full feedback detail

3. PATCH /api/admin/feedback/:id
- allow admin to update:
  - status
  - category
  - adminReply
  - internalNotes
- validate inputs carefully

4. DELETE /api/admin/feedback/:id
- soft delete preferred:
  - set deletedAt and deletedByUserId
- if hard delete is currently easier, only do that if needed, but soft delete is preferred

Optional:
5. POST /api/admin/feedback/:id/restore
- restore soft-deleted feedback

Security:
- every admin feedback route must require admin access
- use the same permanent admin / admin middleware patterns already used elsewhere in the app
- do not trust client-side admin checks
