## Core idea

This app is for musicians to work out whether a single show or a whole tour run is financially worth doing.

It should have **two main modes**:

1. **Single Show Calculator**
2. **Tour Calculator**

The app should feel simple and easy to use, but powerful underneath.

## Main features

### 1. Artist / Band Profiles
Users can create and save profiles such as:
- Solo
- Duo
- Band

Each profile should store:
- profile name
- act type
- home base
- number of people
- default vehicle
- average accommodation cost per night
- average food/per diem per day
- optional notes

Examples:
- Julian James Solo
- Julian James & Beci Kate Duo
- Full Band

### 2. Vehicles
Users can create and save vehicles.

Each vehicle should store:
- vehicle name
- fuel type
- average fuel consumption
- units as L/100km
- optional notes
- optional max passengers

Examples:
- Coaster Bus – diesel – 16 L/100km
- Van – diesel – 9.5 L/100km
- Car – petrol – 7.2 L/100km

### 3. Single Show Calculator
This mode is for quickly checking whether one show is worth doing.

Inputs:
- selected profile
- selected vehicle
- origin
- destination
- return trip yes/no
- distance in km
- fuel price per litre
- show type
- fee or ticket setup
- accommodation cost
- food cost
- other costs
- merch estimate
- trail notes

Outputs:
- estimated fuel used
- fuel cost
- total trip cost
- total income
- net profit/loss
- break-even fee needed
- profit per band member
- status indicator:
  - Worth the Drive
  - Tight Margins
  - Probably Not Worth It

### 4. Tour Calculator
This is a multi-show tour planner.

A tour should include:
- tour name
- selected profile
- selected vehicle
- start location
- optional end location
- return home yes/no
- start date
- end date
- default food/per diem
- trail notes

Inside the tour, the user can add multiple **stops/shows**.

Each stop/show should store:
- date
- town/city
- venue/show name
- show type
- fee or ticketed setup
- merch estimate
- accommodation cost
- extra costs
- fuel price override (optional)
- manual distance override (optional)
- notes

The app should calculate:
- leg distance between each stop
- total distance
- total fuel used
- total fuel cost
- total accommodation
- total food/per diem
- total extra costs
- total show income
- total merch
- total marketing spend
- gross income
- total expenses
- net profit
- profit per show
- profit per day
- profit per band member

Also show a running total as stops are added.

## Show types

Each show can be one of these:

1. **Flat Fee**
2. **Ticketed Show**
3. **Hybrid** (guarantee + % of door)

### Flat Fee
Inputs:
- fee
- merch estimate

### Ticketed Show
Inputs:
- capacity
- ticket price
- expected attendance percentage
- deal type:
  - 100% door
  - percentage split
  - guarantee vs door
- your percentage if relevant
- marketing spend
- suggested marketing spend
- merch estimate

Calculations:
- expected tickets sold
- gross ticket revenue
- your cut
- merch income
- total show income
- break-even tickets
- break-even capacity percentage

### Hybrid
Inputs:
- guarantee
- ticket setup
- split setup
- merch estimate

Calculation:
- user receives the better of guarantee or door deal if that is how the deal is structured

## Marketing feature

This is important.

For ticketed shows, include a **suggested marketing budget**.

Use a simple default recommendation such as:
- suggested marketing spend = 10% to 20% of expected gross ticket revenue
- default recommendation should show around 15%

Example:
suggested marketing spend = expected ticket revenue × 0.15

Allow user to override it manually.

Also show useful outputs such as:
- “You need 28 tickets to break even”
- “You need 40% capacity to break even”
- “At this attendance, you’ll clear $X”
