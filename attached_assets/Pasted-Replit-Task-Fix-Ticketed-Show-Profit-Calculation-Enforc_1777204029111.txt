Replit Task: Fix Ticketed Show Profit Calculation + Enforce Correct Break-Even Logic (Single + Tour)

Priority: CRITICAL (trust-breaking bug)

---

## 🧠 Problem Summary

The app is currently calculating profit incorrectly.

It is doing:
Revenue - Expenses

It is NOT subtracting band/member payouts.

This results in inflated profit and false “covers band fees” messaging.

---

## 🔍 Verified Bug Example (must match this logic)

Inputs:
- Tickets sold: 45
- Ticket price: $55
- Revenue: 45 × 55 = $2,475 ✅
- Expenses: $658.83
- Member payouts: $325

Current (WRONG):
$2,475 - $658.83 = $1,816.17 ❌

Correct:
$2,475 - ($658.83 + $325) = $1,491.17 ✅

The difference ($325) = member payouts not being deducted.

---

## ✅ Required Fix

### 1. Define Costs Properly

baseExpenses =
  fuel + accommodation + venue + production + marketing + other

bandMemberFees =
  total of all member payouts

totalCosts =
  baseExpenses + bandMemberFees

---

### 2. Fix Net Calculation (EVERYWHERE)

Replace ALL net/profit logic with:

net =
  (ticketsSold × ticketPrice)
  + otherIncome (ONLY if explicitly entered)
  - totalCosts

---

### 3. Break-even Logic (KEEP — already correct)

fullBreakEvenTickets =
  ceil(totalCosts / ticketPrice)

baseExpenseBreakEvenTickets =
  ceil(baseExpenses / ticketPrice)

Hero UI:

YOU NEED X TICKETS TO BREAK EVEN
(covers base expenses + band/member fees)

Y tickets covers base expenses only

### 4. Scenario Logic (CRITICAL RULE)

Scenarios must ONLY change ticket count.

DO NOT:
- add expected attendance as money
- add capacity as money
- add scenario % as money

ONLY:

scenarioRevenue = scenarioTickets × ticketPrice

scenarioNet =
  scenarioRevenue
  - totalCosts
  + otherIncome (if explicitly entered)
