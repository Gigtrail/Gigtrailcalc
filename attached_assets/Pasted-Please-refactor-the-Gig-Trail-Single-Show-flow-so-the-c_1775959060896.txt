## 6. Estimate accommodation nights only after Calculate Gig

When the user clicks “Calculate Gig”, estimate accommodation nights during the final calculation.

### Logic

If accommodation_required = no:
- recommended_accommodation_nights = 0
- estimated_accommodation_cost = 0

If accommodation_required = yes:
- calculate total drive time for the run
- include return trip if selected
- determine max driving hours per day:
  - Free users: use fixed default max driving hours per day
  - Pro users: use profile preferred_max_driving_hours_per_day if set, otherwise fallback default

Then calculate:
- driving_days_needed = ceil(total_drive_hours / max_drive_hours_per_day)
- recommended_accommodation_nights = max(0, driving_days_needed - 1)

## 10. Automatic route calculation

When user clicks “Calculate Gig”:

### Required inputs
- Origin location
- Destination location
- Vehicle fuel consumption
- Fuel price
- Return trip toggle

### Route calculation
Use the Google Maps APIs already connected in the app to calculate:
- one-way distance in km
- one-way driving duration
- if return trip is enabled:
  - total distance = one-way distance × 2
  - total duration = one-way duration × 2

### Output fields
Display:
- Distance (km)
- Estimated drive time
- Fuel used (L)
- Fuel cost ($)

If no route can be found:
- show friendly error:
  "We couldn't calculate the route for this run. Please check the locations."

---

## 11. Fuel calculation logic

Use:
fuel_used_litres = (total_distance_km * vehicle_l_per_100km) / 100
fuel_cost = fuel_used_litres * fuel_price

Please ensure:
- all values are numeric
- currency formatting is correct
- no string concatenation bugs

### Example
If:
- total distance = 400 km
- vehicle = 10 L/100km
- fuel price = 2.00

Then:
- fuel used = 40 L
- fuel cost = 80.00
