Use this as a focused redesign prompt for the EXISTING Single Show Calculator screen. This is not a full rebuild from scratch. It is a smart redesign/refactor of the current screen and logic.

GOAL
Redesign the current Single Show Calculator so it feels:
- smoother
- easier to use
- less intimidating
- more interactive
- more visually rewarding
- more likely to drive repeated calc usage

This calculator is the HOOK of the app.

IMPORTANT PRODUCT RULES
1. Free users get 5 calcs per week.
2. A calc is only counted when the user presses the Calculate button and a valid result is returned.
3. Do NOT auto-update results live while users edit fields.
4. Users must be able to use both:
   - sliders / quick controls for exploration
   - hard numeric inputs for exact data
5. If the user changes inputs after the last calc, mark the result as stale and require pressing Calculate again.
6. Add subtle micro-celebration feedback after calculate.
7. Keep real data wiring. Do NOT replace with fake mock data.

CURRENT PROBLEM
The current page is functional, but it still feels like a form. It needs to feel more like a touring decision tool.

The current issues to solve:
- no strong hook at the top
- too much “fill in fields” energy
- too much vertical admin feel
- no result-first reward
- no slider + hard input hybrid controls
- quick view is too weak
- trail notes interrupt flow
- costs are readable but not interactive enough
- no clear stale-result state after edits
- no visible usage motivation near the action

REDESIGN STRATEGY
Keep the existing structure and data model where possible, but improve layout, hierarchy, interaction, and state handling.

  - “Out of date”

Add a subtle result refresh animation:

Do not overdo it.

10. ADD STALE RESULT STATE AFTER ANY EDIT
==================================================

This is critical.

After a result has been generated, if the user changes:
- venue
- date
- income values
- costs
- assumptions
or anything affecting calc

Then:
- mark result strip as stale
- show badge/message:
  - “Out of date”
  - “Inputs changed”
  - “Recalculate to refresh”

Do NOT clear previous result instantly.
Keep it visible but visibly stale.

This protects the 5-calcs-per-week model and prevents live scenario abuse.
