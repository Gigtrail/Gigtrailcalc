Goal:
Fix vehicle → fuel flow and redesign Step 6 so fuel type is derived from vehicle selection, removing duplication and null states.

PART 1 — VEHICLE STEP (STEP 3)

When a vehicle is selected:

- Immediately assign:
  - fuelType (petrol / diesel / LPG / electric)
  - fuelConsumption (e.g. L/100km)

- Persist this data to the tour:
  tour.vehicleId
  tour.fuelType
  tour.fuelConsumption

- If vehicle preset exists, auto-fill these values
- If missing fuel data, block progression with message:
  "This vehicle needs fuel info before continuing"

Remove any need to re-select fuel type later.

  - fuel type dropdown
  - multiple fuel price inputs (petrol/diesel/LPG all at once)
  - “auto-set from vehicle” messaging

- Replace with:
  Single fuel price input based on selected fuelType

Example:
If fuelType = petrol:
  show "Petrol $/L"

Fuel cost should be calculated using:

- fuelConsumption comes from vehicle
