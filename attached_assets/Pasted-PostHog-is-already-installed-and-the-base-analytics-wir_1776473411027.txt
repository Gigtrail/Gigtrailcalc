## 5. VALIDATE EVENT TIMING

Make sure events fire at the correct moment:

- signup_completed → only after account creation succeeds
- login_completed → only after login succeeds
- show_calc_started → when the user genuinely starts the single show flow
- show_calc_completed → only when a valid calculation result is successfully produced
- tour_calc_started → when the user genuinely starts the tour flow
- tour_calc_completed → only when final tour results are successfully produced
- tour_saved → only after the save succeeds
- pro_feature_clicked → when a free user hits a locked feature
- pricing_viewed → when the pricing modal/page actually opens
- upgrade_started → when checkout/subscription flow is launched
- upgrade_completed → only after payment/subscription confirmation succeeds
- calc_error → when a calc genuinely fails
- save_failed → when a save request genuinely fails

Do not fire completion events before success is confirmed.
